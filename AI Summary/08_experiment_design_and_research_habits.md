# 08. 实验设计与科研习惯

## 为什么作业不只是“把代码跑通”

很多人做作业时最容易忽略的一件事是：

> 机器学习不是只会写模型，还要会设计实验、记录过程、比较结果、解释现象。

CS231n 三份作业虽然是课程作业，但实际上已经在训练你一套很重要的科研和工程习惯：

- 先做 baseline
- 再改一个变量
- 看 train / val 曲线
- 解释结果
- 做 error analysis
- 记录超参数和结论

如果把这些习惯整理出来，它们对以后做项目、复现论文、写报告都会非常有帮助。

## 一、实验不是“试试看”，而是“有假设地验证”

一个成熟的实验，不应该只是：

- 改几个参数
- 看看有没有变好

更好的做法是：

1. 先写清楚当前问题
2. 提出一个具体假设
3. 只改一个关键因素
4. 记录变化结果
5. 分析结果是否支持假设

例如：

- 假设：学习率太大导致训练不稳定
- 操作：只把 learning rate 从 `1e-2` 改成 `1e-3`
- 观测：loss 是否更平稳、val accuracy 是否提升

这比“乱调一堆参数”有用得多。

## 二、从 assignment1 到 assignment3 的实验能力升级

## 1. assignment1：最基础的实验闭环

你已经做过这些事：

- 选择学习率
- 调整正则化强度
- 比较 SVM / Softmax
- 看 train / val accuracy

这一阶段最重要的是意识到：

- 训练结果不是单一数字
- 需要比较超参数
- 需要区分训练集和验证集表现

## 2. assignment2：开始关注网络结构和训练稳定性

你会开始比较：

- 有无 batch norm
- 有无 dropout
- 不同初始化
- CNN 结构深浅

这一阶段的实验更偏向：

- 结构选择
- 优化稳定性
- 正则化效果

## 3. assignment3：进入更现代的实验维度

你会开始面对：

- contrastive learning 中增强策略的影响
- CLIP / DINO 特征的读出方式
- diffusion 的 schedule、cfg scale、采样质量
- Transformer 的层数、head 数、embed dim

这一阶段已经很接近真实研究工作了，因为：

- 指标不再只有 accuracy
- 结果有时既要看数值，也要看生成样例或检索效果

## 三、你应该固定掌握的 8 个实验原则

## 1. 先做 baseline

baseline 的意义是：

- 给你一个可靠参考点
- 让后续改动有比较对象

没有 baseline，所有“变好”或“变差”都没有标准。

baseline 可以很简单：

- 最基础 MLP
- 默认学习率
- 不加 fancy trick

## 2. 一次只改一个关键变量

如果你一次改：

- 学习率
- batch size
- 模型结构
- 优化器

最后结果变了，你根本不知道是哪项起作用。

科研中最基本的清晰性要求，就是尽量控制变量。

## 3. 先做“小规模可验证实验”

例如：

- 用更小数据集
- 更少 epoch
- 更小模型
- 单个 batch 过拟合测试

如果小实验都说不清楚，大实验只会更浪费时间。

## 4. 同时看 train 和 val

这点非常关键。

如果只看 train loss，你可能错把过拟合当进步。

最典型的判断：

- train 高，val 也高：欠拟合
- train 低，val 差：过拟合
- train 和 val 都稳步变好：比较健康

## 5. 看曲线，不只看最终数字

loss / accuracy 曲线能告诉你很多最终结果看不出来的东西：

- 是否一开始就爆炸
- 是否后期过拟合
- 是否优化停滞
- 是否学习率调度有效

曲线是动态过程的证据。

## 6. 记录配置和结论

每次实验至少要记录：

- 模型
- 数据
- 学习率
- batch size
- 正则化
- epoch 数
- seed
- 关键结果
- 结论

不记录，过几天你就会忘。

## 7. 做 error analysis

结果差，不要只说“分数低”，要问：

- 错在哪些样本上？
- 是哪些类别混淆？
- 是增强太强还是标签问题？
- 是模型容量不够，还是训练不稳定？

真正有价值的实验结论，来自错误分析而不是单个分数。

## 8. 用 ablation 学习机制

ablation 的价值不只是拿论文图表，而是帮助你理解“哪个模块真的有用”。

例如：

- 去掉 projection head 会怎样？
- 去掉 positional encoding 会怎样？
- 不用 dropout 会怎样？

这类实验最能训练结构理解力。

## 五、不同类型任务的实验重点不一样

## 1. 分类任务

重点看：

- train / val accuracy
- 混淆类别
- 泛化差距

## 2. 序列生成任务

重点看：

- loss
- 生成样例
- 是否重复、是否提前结束、是否语法崩坏

## 3. 检索 / 表示学习任务

重点看：

- top-k retrieval
- kNN 表现
- 表示空间聚类情况

## 4. 生成任务

重点看：

- 样本清晰度
- 多样性
- 条件一致性
- 采样速度和稳定性

这说明实验设计不能只套一个模板，而是要和任务目标对齐。

## 六、你应该如何面对“结果不好看”

这是非常真实的一点。

实验大部分时候不会立刻给你漂亮结果。遇到差结果时，建议按这个顺序处理：

### 1. 确认不是代码 bug

- shape
- loss
- 标签
- 模式切换
- 数据读入

### 2. 确认不是训练配置问题

- 学习率
- batch size
- 初始化
- 正则化

### 3. 再讨论方法本身

很多时候，不是模型思想不行，而是实验没设计好。

### 4. 保留负结果

负结果不是无意义的，它告诉你：

- 哪些方向已经验证不通
- 哪些改动没必要重复试

这本身就是知识。

## 七、科研习惯的一个关键升级：从“调参”到“建立解释”

初学者常见模式是：

- 结果不好
- 改参数
- 结果还是不好
- 再改参数

更成熟的模式是：

- 观察现象
- 提出解释
- 用实验验证解释

例如：

- 现象：train 快速下降，但 val 不涨
- 解释：模型过拟合
- 验证：增大 weight decay 或数据增强，看 val 是否改善

这就是实验真正的科学性。

## 八、三份作业里最值得延续的实验习惯

你可以把它们总结成下面这些长期模板：

- 先用简单方法建立 baseline
- 先保证小样本可过拟合
- 用验证集做选择，不碰测试集
- 画曲线，不只看最终点
- 每次只改一两个关键变量
- 写下结论，不要只记数字
- 保留失败尝试的原因

## 九、以后做更大项目时，这些习惯会如何升级

在更真实的项目中，它们会变成：

- 配置文件管理
- 实验跟踪工具
- checkpoint 管理
- 随机种子复现
- 系统化 ablation
- 统一日志与指标面板

但底层精神和作业阶段是一样的。

## 十、建议你以后专门积累一个“实验笔记库”

可以把每个小项目或作业实验都记录下来，哪怕很简短。

长期来看，这会形成一份非常宝贵的个人资产：

- 你试过什么
- 哪些经验有效
- 哪些错误高频出现
- 你对不同模型的直觉如何发展

## 最后一句总结

实验设计与科研习惯，是把“会写模型”升级成“会做机器学习工作”的关键一步。

如果你把这份能力练出来，以后面对新问题时就不会只是盲目尝试，而会更自然地进入：

> 观察现象、提出假设、设计实验、记录结果、形成解释

这才是真正长期有复利的能力。
