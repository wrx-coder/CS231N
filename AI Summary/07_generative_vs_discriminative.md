# 07. 生成式模型与判别式模型的统一视角

## 为什么这条线很重要

很多学习者会把“分类模型”和“生成模型”完全分开看，觉得它们属于两个世界。

但如果你从概率建模角度看，CS231n 三份作业其实已经把它们放在同一张图上了：

- Softmax、CNN、ViT：更偏判别式
- Captioning：条件生成
- CLIP：相似度对齐，但常用于判别和检索
- DDPM：典型生成式

把这条线理清楚之后，你会更容易理解：

- 为什么不同模型的 loss 长得差很多
- 为什么有的模型只会“判类别”，有的模型可以“造样本”
- 为什么有些任务训练和推理差异特别大

## 一、最核心的概率区分

### 1. 判别式模型

判别式模型关心的是：

\[
p(y \mid x)
\]

也就是：

- 给定输入 \(x\)
- 预测标签 \(y\)

例如图像分类：

\[
x \to \text{class logits} \to p(y \mid x)
\]

### 2. 生成式模型

生成式模型通常关心的是：

\[
p(x)
\quad \text{或} \quad
p(x, y)
\quad \text{或} \quad
p(x \mid c)
\]

也就是：

- 数据本身如何生成
- 或在条件 \(c\) 下如何生成样本 \(x\)

例如 diffusion：

\[
p(x \mid \text{text})
\]

captioning 也可以看成条件生成：

\[
p(\text{caption} \mid \text{image})
\]

## 二、三份作业里的几种典型例子

## 1. 判别式：Softmax / CNN / ViT

这些模型的核心目标是：

\[
\arg\max_y p(y \mid x)
\]

对应的 loss 常常是 cross entropy：

\[
L = -\log p(y \mid x)
\]

特点：

- 输出是类别分布
- 主要看分类性能
- 推理通常比较直接

## 2. 条件生成：Captioning

图像描述模型不是简单分类，而是在生成一句话：

\[
p(w_1, w_2, \dots, w_T \mid \text{image})
=
\prod_{t=1}^{T} p(w_t \mid w_{<t}, \text{image})
\]

这是典型条件生成。

特点：

- 输出是一个序列
- 训练常用 teacher forcing
- 推理要一步一步解码

所以 captioning 虽然也会用交叉熵，但它的任务本质已经不是判别，而是序列生成。

## 3. 表示对齐：SimCLR / CLIP

这类方法不完全等于传统判别式或生成式。

它们更像是在学习一个表示空间，使得：

- 相似对象靠近
- 不相似对象分离

但在使用上，它们常常支持判别性任务：

- zero-shot classification
- retrieval
- nearest-neighbor inference

所以你可以把这类方法看成“介于纯判别和纯生成之外的表示学习路径”。

## 4. 生成式：DDPM

Diffusion 的目标更接近：

- 学习数据分布
- 或学习条件分布 \(p(x \mid c)\)
- 使模型能从噪声逐步生成结构化样本

虽然训练时常表现为噪声回归 MSE，但它服务的是生成过程。

这说明：

> 训练 loss 的表面形式，并不总能直接告诉你模型属于哪类；要看它最终在逼近什么概率对象。

## 三、从任务目标看二者差异

| 维度 | 判别式 | 生成式 |
| --- | --- | --- |
| 主要目标 | 预测标签或决策 | 建模数据或条件生成样本 |
| 典型概率对象 | \(p(y \mid x)\) | \(p(x)\)、\(p(x \mid c)\)、\(p(x,y)\) |
| 输出 | 类别、分数、相似度 | 图像、文本、序列、样本 |
| 推理方式 | 通常一步完成 | 通常逐步采样或逐步解码 |
| 评估方式 | accuracy、AUC、检索召回 | perplexity、FID、样本质量、多样性 |

这个对照表建议你长期记住。

## 四、为什么 captioning 是理解两者区别的好例子

Captioning 很有意思，因为它长得像分类，但本质上是生成。

每一步训练时都像在做分类：

\[
\text{predict next token over vocabulary}
\]

也就是说每个时间步都在做一个 softmax 分类。

但整体上，它优化的是：

\[
p(\text{whole sentence} \mid \text{image})
\]

并且推理时要：

- 从 `<START>` 开始
- 逐词生成
- 把前一步输出喂回模型

所以它是一个很好的中间桥梁，帮助你看到：

- 局部可用判别式损失
- 整体仍然是生成式任务

## 五、为什么 diffusion 看起来不像“直接学 \(p(x)\)”

很多初学者看到 DDPM 会困惑：

- 为什么训练时只是预测噪声？
- 这和生成数据分布有什么关系？

这是因为现代生成模型常常不直接显式写出 \(p(x)\) 的简单形式，而是通过一个更易训练的目标间接建模分布。

对于 diffusion，可以粗略理解成：

- 前向扩散把数据逐步打乱
- 反向过程学习如何一步步恢复结构

因此虽然损失看起来像：

\[
\|\epsilon - \epsilon_\theta(x_t, t, c)\|^2
\]

但最终作用是支撑：

\[
x_T \to x_{T-1} \to \cdots \to x_0
\]

也就是“从噪声采样出数据”。

## 六、从代码结构看，判别式和生成式有哪些明显差别

## 1. 输出头不同

判别式模型常见输出：

- 类别 logits
- 相似度分数
- binary score

生成式模型常见输出：

- 下一个 token logits
- 噪声预测
- 重建图像
- 采样分布参数

## 2. 推理路径不同

判别式常见：

```python
logits = model(x)
pred = logits.argmax(dim=-1)
```

生成式常见：

```python
token = START
for t in range(T):
    logits = model(prefix)
    token = sample_or_argmax(logits[:, -1])
```

或：

```python
x = randn(...)
for t in reversed(range(T)):
    x = reverse_step(x, t, cond)
```

## 3. 训练和推理差异更大

判别式模型的 train/inference 通常比较接近。

生成式模型则常常差异很大：

- captioning：teacher forcing vs autoregressive decoding
- diffusion：随机时间步训练 vs 完整反向采样

这也是为什么生成模型往往更难实现和调试。

## 七、你该如何判断一个新方法更偏哪边

以后学一个新模型时，可以问下面几个问题：

### 1. 它最终输出的是决策，还是样本

- 输出类别或分数，通常更偏判别式
- 输出文本、图像、音频、视频，通常更偏生成式

### 2. 它在逼近什么概率对象

- \(p(y \mid x)\) 更偏判别
- \(p(x)\) 或 \(p(x \mid c)\) 更偏生成

### 3. 它的推理是否需要逐步采样

如果需要多步解码、多步采样，通常更偏生成式。

### 4. 它的表示空间是否主要为“区分”服务，还是为“重建/采样”服务

这会帮助你把对比学习、生成建模、判别学习区分开。

## 八、两种视角各自的工程特点

## 判别式的工程优势

- 训练目标通常更直接
- 评估指标更清晰
- 推理更快
- 更容易在明确标签任务上拿到高精度

## 生成式的工程挑战

- 推理成本更高
- 评价更复杂
- 模式崩塌、样本质量、多样性更难平衡
- 训练与采样路径差异更大

## 生成式的独特价值

- 能创造样本
- 能做条件生成
- 能支持编辑、补全、翻译、描述等开放输出任务
- 往往包含更丰富的数据分布信息

## 九、一个很有用的统一理解：两者并不是绝对割裂

现代模型里，判别和生成经常互相借力。

例如：

- captioning 使用分类式的 token cross entropy 来训练生成任务
- CLIP 的表示可以用于判别，也可支持检索和开放词汇识别
- diffusion 可以加 classifier guidance 或 classifier-free guidance，把判别信息注入生成过程

这说明真正成熟的理解应该是：

> 判别式和生成式是两种主要建模方向，但在现代系统里，它们常常互相融合。

## 十、你以后怎么用这条线帮助自己学习

建议你看到一个新方法时，先写一句话定义它：

- “这是一个判别式图像分类模型”
- “这是一个条件序列生成模型”
- “这是一个跨模态表示对齐模型”
- “这是一个条件扩散生成模型”

这句话会立刻帮助你决定：

- 应该关注什么概率对象
- 应该关注什么 loss
- 应该关注什么评估指标
- 应该关注训练图和推理图的哪些差异

## 最后一句总结

生成式和判别式的区分，不只是课程里的理论分类，而是一种能帮助你快速判断模型目标、代码结构、训练方式和评估方法的高层框架。

只要你以后遇到新模型时先问：

> 它到底是在学“怎么区分”，还是在学“怎么生成”？

很多复杂问题都会一下子清楚很多。
