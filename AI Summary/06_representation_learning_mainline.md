# 06. 表示学习的主线

## 为什么“表示学习”值得单独抽出来

如果只按作业表面去看，你可能会觉得自己学的是很多分散任务：

- 分类
- 图像描述
- 检索
- 分割
- 生成

但从更高一层看，这些任务都在围绕同一个核心问题打转：

> 如何把原始输入变成一种有用、稳定、可迁移的内部表示？

这就是表示学习。

所谓“表示”，可以理解为模型内部对数据的一种压缩、组织和抽象方式。它决定了模型最终能不能：

- 分类
- 检索
- 迁移
- 对齐不同模态
- 支撑生成

CS231n 三份作业里，其实已经让你碰到了表示学习的几条重要主线。

## 一、什么叫“好表示”

一个好的表示，不是简单地“维度高”或者“看起来复杂”，而是满足下面这些性质中的若干条：

- 对任务有判别性
- 对无关扰动有不变性
- 对相似样本有邻近性
- 对不同任务有迁移性
- 在下游任务中容易被线性头读出

可以把目标写成一种非常粗略的直觉：

\[
x \xrightarrow{f_\theta} z
\]

其中 \(z\) 既要保留重要信息，又要丢掉不重要的噪声。

这听起来简单，但不同方法对“重要信息”的定义完全不同。

## 二、assignment1：表示学习还很弱，但雏形已经出现

assignment1 里最开始是：

- kNN：几乎没有表示学习，只是在原空间或手工特征空间做距离比较
- 线性分类器：学习一组线性判别边界
- MLP：开始通过隐藏层学习中间表示

### 1. 手工特征阶段

像 HOG、HSV histogram，本质上是在手工设计表示：

\[
x \mapsto \phi(x)
\]

这类方法的思想是：

- 人工把“有用模式”写进特征提取器
- 再让简单分类器去使用这些特征

### 2. MLP 阶段

两层网络开始自己学习表示：

\[
h = \mathrm{ReLU}(XW_1+b_1)
\]

这里的 \(h\) 就是隐藏表示。

这一步非常关键，因为它意味着：

- 模型不再只在原始像素上分类
- 而是先把输入变成对任务更有利的内部表示

## 三、assignment2：表示开始和结构偏置结合

assignment2 引入 CNN 和 RNN 后，表示学习开始明显依赖结构先验。

## 1. CNN 表示

CNN 学到的是层级化的视觉表示：

- 低层：边缘、纹理
- 中层：局部形状和部件
- 高层：对象级语义

这是因为卷积天然引入了：

- 局部连接
- 权重共享
- 平移等变性

所以 CNN 的表示很适合图像这种有强空间结构的数据。

## 2. RNN 表示

RNN 学的是序列表示：

- 当前状态不仅编码当前 token
- 还压缩了过去上下文

形式上：

\[
h_t = f(x_t, h_{t-1})
\]

所以 \(h_t\) 是一个条件于历史的表示。

这说明表示学习不是只有“图像特征”，也包括“上下文状态表示”。

## 四、assignment3：表示学习的几条现代主线

assignment3 是真正把你带进现代表示学习世界的部分。

## 1. Supervised representation

最经典的形式是分类训练：

\[
z = f_\theta(x), \quad logits = Wz
\]

通过交叉熵训练，让表示对类别可分。

优点：

- 目标明确
- 在标签任务上效果强

局限：

- 依赖人工标签
- 学到的表示更偏向当前标签体系

ViT 分类其实仍然是这一主线的延续。

## 2. Contrastive representation：SimCLR

SimCLR 的关键思想是：

- 同一张图的两次增强视图应该表示接近
- 不同图像应该在表示空间中区分开

形式上：

\[
z_i = f_\theta(\mathrm{aug}_1(x)), \quad
z_j = f_\theta(\mathrm{aug}_2(x))
\]

希望正样本对接近：

\[
\mathrm{sim}(z_i, z_j) \text{ 大}
\]

负样本对分离。

这条主线非常重要，因为它代表：

> 不靠人工标签，也能通过结构化目标学表示。

## 3. Cross-modal representation：CLIP

CLIP 更进一步，不只是把同图像的两种增强拉近，而是把“图像”和“文本”拉到同一空间：

\[
v = f_{\text{img}}(x), \quad
t = f_{\text{text}}(y)
\]

\[
\mathrm{sim}(v, t) \text{ 对应语义匹配程度}
\]

这意味着表示不再只是“视觉内部表示”，而是：

- 可跨模态对齐
- 可做 zero-shot 分类
- 可做检索

CLIP 让你看到一种非常现代的表示观：

> 好表示不仅要能分类，还要能和别的模态建立语义对应。

## 4. Self-supervised dense representation：DINO

DINO 让你接触到另一种有意思的表示：

- 不只是图像级 embedding
- 还有 patch 级、局部区域级的语义表示

这在 dense task 中尤其有用，例如分割。

也就是说，表示学习不仅分“学什么语义”，还分“在什么粒度上表达语义”：

- 图像级
- patch 级
- token 级
- 像素级

## 5. Generative representation：Diffusion 的隐含表示

虽然 DDPM 主要是生成模型，但它也在学一种表示：

- 给定带噪图像、时间步和条件
- 网络内部必须形成对图像结构和噪声结构的联合表征

这种表示和分类表示不同：

- 它更关注数据分布与去噪过程
- 不一定直接为“线性分类”服务

这提醒你：

> 表示学习并不只服务于判别任务，生成任务也在塑造内部表示空间。

## 五、不同表示学习目标在本质上优化什么

## 1. 分类目标

优化：

\[
p(y \mid x)
\]

强调：

- 类别可分
- 决策边界清晰

## 2. 对比目标

优化：

- 正样本相似
- 负样本分离

强调：

- 相对关系
- 几何结构
- 增强不变性

## 3. 跨模态对齐目标

优化：

- 图像表示和文本表示共享语义空间

强调：

- 语义对齐
- 模态之间的可比较性

## 4. 生成目标

优化：

- 对数据分布的建模能力

强调：

- 数据重建或采样
- 对输入结构的细致建模

## 六、你应该如何比较不同表示

一个很实用的方法是用下面几个维度对比：

| 维度 | 可以问的问题 |
| --- | --- |
| 监督来源 | 是人工标签、数据增强、图文对，还是纯自监督？ |
| 粒度 | 图像级、patch 级、token 级还是像素级？ |
| 不变性 | 它希望对哪些变化不敏感？ |
| 可迁移性 | 是否容易迁移到别的下游任务？ |
| 读出方式 | 线性分类、最近邻、检索、生成，哪个最自然？ |

这张表会帮助你避免“只看模型名，不看表示本质”。

## 七、从 CS231n 三份作业里能提炼出的表示学习主线

你可以把整条线概括成下面这样：

### 第一阶段：人工定义表示

- HOG
- 颜色直方图

### 第二阶段：监督学习表示

- MLP
- CNN
- ViT 分类

### 第三阶段：序列上下文表示

- RNN hidden states
- Transformer token states

### 第四阶段：对比式表示

- SimCLR

### 第五阶段：跨模态表示

- CLIP

### 第六阶段：稠密自监督表示

- DINO patch features

### 第七阶段：生成式内部表示

- DDPM / U-Net 中的层级去噪表示

这是一个很漂亮的演化路径。

## 八、以后你该怎么判断“这个方法到底学到了什么表示”

建议你固定问下面几个问题：

### 1. 输入经过什么变换进入表示空间

- augmentation？
- tokenization？
- patchify？

### 2. 模型输出的哪个层是真正拿来用的表示

- backbone 最后一层？
- projection head 前？
- projection head 后？
- CLS token？
- mean pooling？
- patch tokens？

### 3. loss 在鼓励表示具备什么性质

- 类别可分？
- 视图不变性？
- 跨模态对齐？
- 去噪一致性？

### 4. 下游任务如何读取这个表示

- 线性分类
- kNN
- 检索
- 生成
- 分割

这几问会帮助你真正抓住方法精髓。

## 九、你可以如何继续扩展这条主线

如果以后继续学，表示学习可以沿这些方向往下走：

- metric learning
- masked autoencoding
- BYOL / SimSiam
- language model hidden states
- multimodal foundation model representations
- latent diffusion 中的潜空间表示

这样你会发现，很多现代方法其实都仍然围绕“学好表示”在做文章。

## 最后一句总结

表示学习是把分散任务串起来的一条主线。

当你以后再学新方法时，如果能先问自己：

> 它到底在学一种什么样的表示？这个表示希望保留什么、忽略什么、服务什么下游任务？

那你对模型的理解就会一下子上升一个层次。
